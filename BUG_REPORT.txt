# تقرير تحليل مشكلة الاتصال في تطبيق Fox Call

## تاريخ التقرير: 2026-04-25

---

## المشكلة الأساسية

عند محاولة إجراء مكالمة، التطبيق لا ينجح في الاتصال رغم أن كل شيء آخر يعمل بشكل صحيح (تسجيل الدخول، عرض الرصيد، etc.)

---

## أسباب المشكلة

### 1. مشكلة في تهيئة صوت Linphone قبل بدء المكالمة

في ملف `LinphoneCallModule.kt`، الـ audio focus يُطلب قبل بدء المكالمة، لكن هذا قد لا يكون كافياً. Linphone يحتاج لتهيئة كاملة للصوت قبل أن يرسل SIP INVITE.

**السطر المسؤول:** ~235-247 في `configureAudioForCall()` و `requestAudioFocusBeforeCall()`

### 2. مشكلة في معالجة أخطاء SIP

عند فشل المكالمة، التطبيق لا يعرض رسائل خطأ واضحة للمستخدم. الـ error parsing قد لا يلتقط كل الحالات.

**السطر المسؤول:** ~265-279 في `parseCallError()`

### 3. عدم وجود قائمة للحسابات المستخدمة/الفاشلة في التطبيق

التطبيق لا يعرف إذا كان الحساب (SIP credentials) تم استخدامه قبل كده أو أن رصيده خلص أو الحساب معطل.

---

## الحل المطلوب

### 1. تحسين تهيئة الصوت
- إضافة تأخير قصير (500ms) بعد طلب audio focus
- تحسين اختيار أجهزة الصوت (input/output)
- إضافة retry mechanism للاتصال

### 2. إضافة قائمة الحسابات المستخدمة
- تتبع الحسابات اللي رصيدها خلص
- تتبع الحسابات اللي اتستخدمت قبل كده
- عرض هذه المعلومات للمستخدم

### 3. تحسين معالجة الأخطاء
- ترجمة جميع رسائل الخطأ للعربية
- إضافة error codes واضحة

---

## الملفات المتأثرة

1. `/modules/linphone-call/android/src/main/java/expo/modules/linphonecall/LinphoneCallModule.kt`
2. `/services/callManager.ts` (فريموورك)
3. `/screens/TokenScreen.tsx` (لعرض حالة الحساب)

## خطوات الإصلاح

### الخطوة 1: تحسين LinphoneCallModule.kt
- إضافة delay بعد audio focus
- تحسين error handling
- إضافة retry logic

### الخطوة 2: إضافة نظام تتبع الحسابات المستخدمة
- إضافة API endpoint جديد للجوال
- عرض الحسابات المستخدمة في قائمة

### الخطوة 3: تحسين رسائل الخطأ
- ترجمة جميع الحالات
- عرض نصي واضح للمستخدم

## ملاحظة مهمة

التطبيق يستخدم EAS Build أو Expo Go. للـ native modules (Linphone)، لازم يعمل build محلي أو يستخدم `expo run:android` لأن Expo Go مش بيدعم native modules.
